- **项目名称**: QuotaX - B2B 报价展示工具

- **文档版本**: 1.0
- **更新日期**: 2026-03-10

---

## 1. 产品概述

### 1.1 产品背景

B2B 供应商目前主要使用 Excel/PDF 向客户发送报价，存在以下问题：
- 制作耗时（30-60分钟/份）
- 展示不专业，客户体验差
- 无法追踪客户是否查看
- 移动端查看体验极差

### 1.2 产品目标

打造一款轻量级 B2B 报价展示工具，让供应商：
- 3分钟内生成专业报价单
- 提供类似淘宝的购物体验
- 实时追踪客户行为
- 支持移动端完美展示

### 1.3 目标用户

**主要用户**：
- 批发商、经销商（年营业额 100万-5000万）
- 工业品供应商
- 建材、五金、电子元器件供应商

**用户特征**：
- 年龄：25-45岁
- 数字化程度：中等
- 痛点：报价效率低、展示不专业
- 付费意愿：中高

## 3. 详细功能设计

### 3.1 用户系统

#### 3.1.1 微信登录

**流程**：
1. 用户点击"微信登录"
2. 调用微信登录接口获取 code
3. 后端用 code 换取 openid 和 session_key
4. 创建用户记录（新用户）或返回用户信息（老用户）
5. 返回 JWT token

**接口**：
```
POST /api/auth/login
Request: { code: string }
Response: { token: string, user: UserInfo }
```

#### 3.1.2 基础信息设置

**字段**：
- 公司名称（必填）
- 联系人（必填）
- 联系电话（必填）
- 公司Logo（选填）
- 公司介绍（选填）

---

### 3.2 产品管理

#### 3.2.1 产品模型

```typescript
interface Product {
  id: string;
  name: string;           // 产品名称
  description?: string;   // 产品描述
  images: string[];       // 产品图片
  categoryId: string;     // 分类ID
  price: number;          // 价格
  priceType: 'retail' | 'wholesale' | 'agent';  // 价格类型
  specs: Spec[];          // 规格参数
  unit: string;           // 单位
  stock?: number;         // 库存
  status: 'on' | 'off';   // 上下架状态
  sort: number;           // 排序
  createdAt: Date;
  updatedAt: Date;
}

interface Spec {
  name: string;   // 规格名称
  value: string;  // 规格值
}
```

#### 3.2.2 产品列表

**功能**：
- 分页展示（每页 20 条）
- 支持按分类筛选
- 支持按状态筛选
- 支持搜索
- 支持排序（创建时间、价格、名称）

#### 3.2.3 添加/编辑产品

**表单字段**：
- 产品名称（必填，最多 50 字）
- 产品图片（必填，最多 5 张）
- 产品分类（必填）
- 产品价格（必填，数字）
- 价格类型（单选：零售价/批发价/代理价）
- 产品规格（可选，键值对形式）
- 单位（必填，如：件、个、箱）
- 产品描述（可选，富文本）

#### 3.2.4 分类管理

**功能**：
- 创建分类（名称、排序）
- 编辑分类
- 删除分类（分类下有产品时禁止删除）
- 拖拽排序

**分类模型**：
```typescript
interface Category {
  id: string;
  name: string;
  sort: number;
  productCount: number;
  createdAt: Date;
}
```

### 3.3 报价展示

#### 3.3.1 报价单页面

**页面结构**：
```
- 顶部：公司信息（Logo、名称、联系方式）
- 搜索栏：关键词搜索
- 分类栏：横向滚动分类
- 产品列表：瀑布流/网格布局
  - 产品卡片：图片、名称、价格、规格预览
- 底部：购物车入口
```

**产品卡片设计**：
- 图片：正方形，占卡片 60%
- 名称：最多 2 行，超出省略
- 价格：大号字体，红色突出
- 规格：最多显示 2 个关键规格
- 操作：加入购物车、收藏

#### 3.3.2 产品详情页

**页面结构**：
```
- 图片轮播
- 产品名称
- 价格
- 规格参数（表格形式）
- 产品描述
- 底部操作栏：加入购物车、立即询价
```

## 4. 非功能需求

### 4.1 性能要求

- 页面首屏加载 < 2s
- 接口响应时间 < 500ms
- 图片懒加载
- 列表虚拟滚动（产品数 > 100 时）

### 4.2 兼容性

- 微信小程序基础库 >= 2.19.0
- 支持 iOS 10+、Android 5.0+
- 适配各种屏幕尺寸

### 4.3 安全要求

- 接口使用 HTTPS
- 用户认证使用 JWT
- 敏感操作需要二次确认
- 数据权限隔离（用户只能访问自己的数据）

### 4.4 可用性

- 系统可用性 >= 99.5%
- 数据备份策略（每日备份）
- 故障恢复时间 < 30分钟

## 5. 界面设计

### 5.1 设计原则

- **简洁**：减少不必要的元素
- **直观**：用户无需学习即可使用
- **美观**：参考淘宝、京东等电商 APP
- **品牌感**：支持自定义 Logo 和配色

### 5.2 色彩规范

**主色调**：
- 品牌色：#1890ff（蓝色）
- 成功色：#52c41a（绿色）
- 警告色：#faad14（橙色）
- 错误色：#f5222d（红色）

**中性色**：
- 标题：#262626
- 正文：#595959
- 辅助文字：#8c8c8c
- 边框：#d9d9d9
- 背景：#f5f5f5

### 5.3 字体规范

- 标题：18px，加粗
- 正文：14px
- 辅助文字：12px
- 价格：16px，加粗，红色

## 6. 数据埋点

### 6.1 核心指标

| 指标 | 说明 | 计算方式 |
|-----|------|---------|
| 活跃用户数 | 日/周/月活跃用户 | 登录用户数 |
| 产品创建数 | 用户创建的产品数 | 产品表统计 |
| 报价单查看数 | 报价单被查看次数 | 访问日志 |
| 购物车转化率 | 加购到导出的转化率 | 导出数/加购数 |
| 分享次数 | 报价单被分享次数 | 分享日志 |

### 6.2 埋点事件

| 事件 | 触发时机 | 参数 |
|-----|---------|------|
| login | 用户登录 | userId, source |
| product_create | 创建产品 | userId, productId |
| product_view | 查看产品 | userId, productId |
| cart_add | 加入购物车 | userId, productId, quantity |
| cart_remove | 从购物车移除 | userId, productId |
| favorite_add | 收藏产品 | userId, productId |
| quotation_share | 分享报价单 | userId, shareType |
| order_export | 导出订货单 | userId, format |
